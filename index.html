1. 产品概述

REF Life Cycle Platform 以租赁项目为主线，连接空间规划、租赁交易、项目交付、设施运营和成本管理，解决跨团队信息分散、流程断点、重复录入以及合同结果偏离决策基线等问题。

核心目标

  • 统一贯穿需求、寻源、评估、决策、立项、合同、付款、交付、运营与退出。
  • 上游字段自动传递，并显示来源、版本和同步时间。
  • 支持同一楼宇多合同、同一合同多楼层。
  • 合同最终值与选址评估、决策和立项基线自动比对。
  • 支持中英文和响应式访问。

4. 租赁生命周期

新租 / 扩租

需求确认 → 场地寻源 → 选址评估 → 选址确认 → 业务场地确认 → 场地信息登记 → 租赁立项 → CS 邮件同步 → 合同签署 → 拿场地 → Target Day 1

续租 / 换租

续租可选择继续续租或转为换租。换租不建立第三种流程，而是派生独立的新租项目和退租项目，通过 relation_group_id 双向关联。

退租

退租需求 → 范围确认 → 业务场地确认 → 退租审批 → 正式通知 → 复原与交还 → 费用结算 → 台账关闭

5. 选址评估

楼宇合规评估

位于短名单评估，不在推荐场地决策材料中重复出现。技术风险移动到设计评估。

  5. 总租金

  7. 押金总额
  8. 押金说明（押金标准，是否可保函）
  9. 复原费
  10. 其他
  11. 预估装修单方造价
  12. 预估项目总成本
  13. 施工是否指定供应商
  14. 全周期单工位成本

设计评估 15 项

7. 租赁合同管理

页面结构

  1. 按楼宇查看租赁合同。
  2. 楼宇内切换“按合同查看 / 按楼层查看”。
  3. 下方保留全量租赁合同台账。

不展示右侧“合同系统联动”纯说明板块。

楼宇、合同与楼层

  • 同一楼宇允许多份合同，例如首租 L2–5、扩租 L6–10。
  • 按合同查看展示楼层、面积、类型、期限、状态和关键提醒。
  • 按楼层查看必须为表格列表，每层一行，不使用卡片模块。
  • 楼层列表字段：楼层、对应合同、合同类型、分摊面积、期限、状态、操作。
  • 全量台账字段：合同编号、项目/场地、类型、楼层、租期、状态、提醒、操作。

8. 合同决策基线偏差预警

合同继承字段保存上游快照，包括字段编码、原始值、标准化值、单位、来源对象、来源版本和同步时间。合同修改不得覆盖上游历史。

首批比较字段

租赁面积、起止日期、租期、表面/有效租金、物业费、租金与物业总额、保证金、单工位成本及关键商务条款。

交互规则

  • 编辑过程中实时比较，保存后服务端再次校验。
  • 不一致时字段容器、字段名和合同值高亮。
  • 字段旁显示原决策数据及来源。
  • 详情顶部汇总差异数量。
  • 高风险差异可阻断合同提交或签署，直至有权限人员确认。
字段原决策数据合同数据结果
合同单工位成本¥6,820 / 工位 / 月¥7,000 / 工位 / 月高亮预警并显示原值
比较必须使用标准化值：金额统一币种和精度、日期统一 ISO、面积统一平方米、枚举比较 code。允许配置提示容差和阻断阈值。

11. 状态机与校验

选址确认

DECISION_DOCUMENT_PENDING → BUSINESS_CONFIRMATION_READY → SITE_REGISTRATION_READY → INITIATION_READY → CS_SYNC_READY → COMPLETED

合同

DRAFT → TERMS_NEGOTIATION → APPROVAL → SIGNING → SIGNED → ARCHIVED

进入审批或签署前必须重新计算偏差、阻断未确认的高风险项，并固化合同与决策基线版本。

12. 前端实现要求

  • 按路由、领域组件、服务层、权限层和状态机拆分，不沿用单文件原型结构。
  • 使用字段 code、单位、来源和基线，不通过显示文案判断业务。
  • 编辑采用明确的编辑、保存、取消模式。
  • 楼宇、合同、楼层视图共享查询状态，返回后恢复选中楼宇和视图。
  • 中英文使用 i18n key，货币、面积、日期按 locale 格式化。
  • 偏差状态同时使用图标、颜色和文字，满足无障碍要求。

13. 非功能要求

  • 桌面首屏 LCP <2.5s，普通列表 API P95 <800ms。
  • 关键写接口可用性 ≥99.9%，具备重试和幂等保护。
  • 满足 WCAG 2.1 AA，支持键盘、可见焦点和屏幕阅读器。
  • 敏感字段加密，附件校验 MIME、大小、病毒和权限。
  • 字段、状态、审批、文件、发送、AI 建议和偏差确认全部审计。

14. P0 验收标准

  • 支持新租、扩租、续租、退租及换租双项目。
  • 楼宇合规位于短名单评估，成本和设计字段符合指定顺序。
  • 选址确认仅显示五步，场地登记展开位于第 3 步下方。
  • 合同可按楼宇、合同和逐层表格查看，并保留全量台账。
  • 同一楼宇支持多合同，同一合同支持多楼层。
  • 合同值偏离决策基线时实时高亮，显示原值、来源和差异数量。
  • 高风险偏差可阻断提交或签署。
  • 中英文切换不丢失页面、表单和筛选状态。
  • 所有 P0 写操作覆盖权限、校验、并发和异常场景。

15. 建议实施顺序

  1. 项目、楼宇、楼层、合同和字段字典。
  2. 寻源、短名单和专业评估。
  3. 选址确认五步状态机与文档存档。
  4. 租赁立项和上游字段快照。
  5. 楼宇 / 合同 / 楼层三种合同查询视图。
  6. 合同详情编辑、基线比较和偏差确认。
  7. 付款、提醒和组合看板。
  8. 权限、审计、国际化、性能和可访问性。
原型中的数据、AI 回复、上传、审批、邮件、保存、偏差和第三方集成均为前端模拟。生产环境必须由服务端执行权限、状态机、偏差计算、审计和幂等校验。
